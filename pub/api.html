Swish.js

Create interactive basketball shot charts and simulation tools with this small javascript library

API | Documentation
Getting Started

Load Swish.js into your webpage by adding the following to the head of your HTML file:

  • swish.css containing styles for the library
  • jQuery from the Google CDN
  • leader-line.min.js for an external library, LeaderLine
  • swish.js

Note: LeaderLine is used to create the arrows in the visualizations

<‍link rel="stylesheet" type="text/css" href="css/swish.css">

Data

The data passed into function drawActions()

Required data:

  • type: The action types including make, miss, pass, dribble, rebound, steal, block, turnover, and foul. Also accepts new types
  • x: The x coordinate of the action which must have a value between 0-50 for a half or full court
  • y: The y coordinate of the action which must have a value between 0-47 for a half court and 0-94 for full court

Optional data:

  • x2: The end x coordinate of the actions with type "pass" or "dribble". Required data if type is "pass" or "dribble"
  • y2: The end y coordinate of the actions with type "pass" or "dribble". Required data if type is "pass" or "dribble"
  • player: The player that performed the action
  • team: The team of the player that performed the action. The data can contain maximum of 2 teams
  • period: The period/quarter of the basketball game
  • time: The time left in the period/quarter of the basketball game
  • description: The description of the action

const data = [

Functions
.drawCourt(courtID, width, courtType, divID)

Draws a court and places it in a <‍div> element with the id, "divID". A new "courtID" is required for each court. The court will have a width of "width". The court will be a halfcourt if courtType equals "half" and a fullcourt if courtType equals "full".

.drawActions(data)

Draws the actions in "data". Data requirements can be found above.

  • Scales the size and position of the actions before plotting on the court
  • Default colours for each of the predefined action types and generates a random colour for new types
  • Default circle shape to show actions
  • Default solid arrows to show actions of type "pass" or "dribble"
  • If there are teams in the data, circles and solid arrows will be assigned to the first team in the data, and squares and dashed arrows will be assigned to the second team in the data
.createSlider(divID)

Creates an interactive slider and places it in a <‍div> element with the id, "divID".

Drag the slider to hide or show the different actions in the order that they are in the data. The slider will have a minimum of 0 to hide all the actions on the court and a maximum of the length of the data to show all of the actions on the court.

The current action will also be highlighted in light blue.

.createTimeline(width, height, divID)

Creates a timeline and places it in a <‍div> element with the id, "divID". The timeline will have width of "width" and height of "height".

The timeline will display the period/quarter, the time, and the description of each action in the data.

If a slider was previously created, dragging the slider will also update the timeline with the current action. The details for the current action in the timeline will also be highlighted in light blue.

.createPopup()

Creates a popup for each action. The popup displays when the user hovers over the action.

The popup will display the player that performed the action, their team, and the type of the action.

If there is no data on the team or player, the popup will only display the type of the action.

.setColour(type, colour)

Sets the action of type, "type", to a new colour.

.setShape(team, shape)

Sets the team, "team", to a new shape. Shapes can be "circle" or "square".
